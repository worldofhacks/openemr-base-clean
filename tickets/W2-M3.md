## Context

W2-D2 locks LangGraph as the router; the W1 direct loop
(`agent/app/orchestrator/loop.py` — `Orchestrator`, `ToolRegistry`, `BriefResult`)
survives **unchanged inside a worker node**. `langgraph>=1.2,<2` is pre-staged in
`agent/pyproject.toml` (orchestrator commit `5c4e5ef`): this ticket must NOT touch
pyproject or the Dockerfile (W2-M1 owns them this wave). The current `/chat` route
(`agent/app/routes/chat.py`) is a JSON POST: it resolves the pinned session via the
`ChatService` protocol on `app.state.services`, enforces the cross-patient refusal, calls
`run_brief`, and returns `ChatResponse` — today there is no SSE in code; §2a's SSE
contract is exactly what this spike probes behind the flag.

Design (proposed, keeping the dispatched file scopes exact): the flag is env var
`W2_GRAPH_ENABLED` (default unset = OFF), read by a small helper in
`agent/app/orchestrator/graph.py` and consumed by `chat.py` — with the flag OFF, W1
behavior is **bit-identical** (no graph module in the execution path); promotion of the
flag into `app/config.py` settings belongs to a later feature ticket. `state.py` owns the
per-turn typed graph state plus `HandoffRecord` and its **closed enums** as Pydantic v2
strict models (`extra="forbid"`): `HandoffRecord{correlation_id, turn,
supervisor_decision, reason_code, worker, input_ref, output_ref, handoff_ts}` with
`supervisor_decision ∈ {route_extract, route_retrieve, compose_answer, refuse, done}` and
a closed reason_code enum per decision (§2 — locked-decision). Graph state is constructed
per turn and discarded at turn end — **no LangGraph checkpointer** (§2, locked-decision).
The per-turn step budget uses the §2 working value **8**; exhaustion emits a terminal
HandoffRecord (`reason_code=step_budget_exceeded`) surfacing as a W1-canonical refusal.
The two stub workers (`stub_extractor.py`, `stub_retriever.py`) are placeholders replaced
by W2-M9/M14 — no real extraction or retrieval here.

Deterministic tests reuse the existing seams (read `agent/tests/conftest.py` +
neighbors): the `complete_env` fixture; fake LLM providers as in
`agent/tests/test_chat_route.py` (`_SubmitClaimsProvider`) and
`agent/tests/test_verify_in_loop.py`; the observability sink seam
(`InMemoryTraceSink` in `agent/app/observability/langfuse.py`, exercised in
`agent/tests/test_observability_sink.py`) and the fake-Langfuse observation capture
pattern in `agent/tests/test_langfuse_sink_v4.py` for span parent-child assertions;
services injected on `app.state.services` so no live OpenEMR/Anthropic is needed.

## Acceptance Criteria

- **AC-1**: Given the graph flag ON and a fake LLM provider, when a turn routes through
  the supervisor to each of the two stub workers, then **every hop emits a
  `HandoffRecord`** whose `supervisor_decision` and `reason_code` are members of the
  closed enums, with `correlation_id`, `turn`, `worker`, `input_ref`, `output_ref`, and
  `handoff_ts` populated, and unknown/extra fields rejected (strict Pydantic).
- **AC-2**: Given the observability sink seam capturing spans, when a graph turn runs,
  then the supervisor span is the **parent of the worker spans** (parent-child ids
  asserted against the captured observations) and the hop sequence is reconstructable
  from the correlation ID alone.
- **AC-3**: Given the same fake LLM input, when the same request runs through the W1
  direct loop and through the graph's worker embedding of `loop.py`, then the answer
  output is identical (same fake in, same answer out — W1-loop-in-worker equivalence).
- **AC-4**: Given default settings (flag unset/OFF), when `POST /chat` is exercised via
  TestClient with injected fake services, then behavior is **bit-identical to W1** —
  identical response payload for identical inputs, all W1 session/refusal error mappings
  intact (404/401/503/403), and the graph entrypoint provably not invoked (e.g.
  monkeypatched to raise if called).
- **AC-5**: Given an intentionally-looping stub graph, when it runs under the recursion/
  step budget (working value 8), then it terminates via budget exhaustion — a terminal
  `HandoffRecord` with `reason_code=step_budget_exceeded` surfacing as a W1-canonical
  refusal — within bounded test wall-clock, never a hang.
- **AC-6**: Given the graph flag ON, when a TestClient request opts into streaming on the
  `/chat` surface, then an SSE stream (`text/event-stream`) is served from the graph
  path: it yields at least one content event and a terminal event, and the connection
  closes cleanly (fake provider; deterministic).
- **AC-7** [live-measure]: Given a live traced run, when the trace is inspected in
  Langfuse, then the REAL span nesting shows supervisor span ⊃ worker spans and the flow
  reconstructs from the correlation ID alone — screenshot/trace-id evidence recorded in
  the ticket report (W1 D16 content-OFF posture unchanged).
- **AC-8** [live-measure]: The SSE token-streaming **verdict is recorded**: token
  streaming through LangGraph workers works, OR the named §2a fallback is invoked —
  stream only the final composer stage — with its perceived-latency cost noted for the
  cost report (never a correctness cost).
