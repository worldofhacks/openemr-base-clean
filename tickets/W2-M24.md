## Context

W2-D8 locks the two-tier gate: Tier 2 is the full **50-case live-Anthropic run**
(PR-blocking in GH Actions) whose per-case unit is **three calls** — VLM extraction +
answer turn + pinned-judge turn — with **multi-page VLM calls counted explicitly, never
hidden inside "50 turns"**, and retries counted as amplification. The architecture retired
the "$4/run" claim (§7 SUPERSEDED): a spike must measure the real bound **before Tier 2
becomes required**. This ticket is that spike, plus the frozen fork-PR secret policy
W2-M20 consumes.

**Key facts:** the fork repo `worldofhacks/openemr-base-clean` GitHub Actions secrets are
EMPTY (W2-OA2 pending — noted, NOT blocking). The live measurement therefore runs against
the **local agent key from `agent/.env`** (`ANTHROPIC_API_KEY`; model from `LLM_MODEL`)
and **must NOTE this substitution in its output**. Secret VALUES are never read into
reports, printed, logged, or committed — the tooling reads env var names only and treats
values as opaque. Sample inputs are **tiny synthetic non-clinical images generated at
runtime** (throwaway; provider outputs are not committed). Image-generation path
(pre-authorized — this ticket may add NO dependencies; `agent/pyproject.toml` is
W2-M1-owned): generate the image bytes **stdlib-only** (e.g. a minimal valid PNG via
`zlib` + `struct`); alternatively, if W2-M1 has already merged to `swarm/w2-wave0` by
the time AC-7's live run executes, pillow (installed transitively via pdfplumber) may be
used. Either way the frozen tests AC-1..AC-6 are unaffected — they are offline with
provider calls faked.

`agent/ops/spike_tier2.py` is a NEW operator measurement CLI (ops scripts may print —
`.tdd-swarm/gates.md`). It runs a small representative sample (proposed working size:
5 units, per the plan's "representative sample") of the real three-call-unit shape,
records per-call-class runtime/tokens/cost/rate-limit headroom/retry amplification,
extrapolates with the explicit formula **`50 × (VLM extraction + answer turn + judge)`**,
names a maximum per-run cost/time budget, and states the viability verdict for making the
full 50-case gate required. **Failure to fit quota/runtime/cost is a STOP escalation for
making Tier 2 required — never solved by reducing the 50 cases or bypassing the gate**
(locked-decision).

`docs/week2/W2_TIER2_CI_POLICY.md` is a NEW policy note (allowed exception: a new W2_*
doc — never an edit of a binding doc; `W2_DEVLOG.md` is orchestrator-owned and off-limits)
that **freezes** (locked-decision, from §6a + W2-D8): forks get NO repository secrets;
fork code is never checked out under `pull_request_target`; forks run Tier 1 only, and a
maintainer must reproduce the exact fork commit on a trusted same-repository branch for
the required Tier-2 result before merge (any new commit invalidates approval/status);
same-repo PRs use least-privilege environments with approval and no secret
echo/artifact retention. The policy lint asserts read-only over the EXISTING
`.github/workflows/*.yml` (reads are fine; writes to workflows are out of scope).

**Known compliant near-miss the lint must pass:**
`.github/workflows/dependabot-auto-merge.yml` triggers on `pull_request_target` AND
consumes `secrets.AUTO_MERGE_APP_PRIVATE_KEY` — but performs **no checkout of PR code**,
so it satisfies the three-way conjunction (trigger + checkout-ref + secret usage) and is
compliant. A naively over-broad lint (`pull_request_target` + secrets = violation) would
permanently red the frozen test against a file this ticket is forbidden to modify. The
test author must handle absence-of-checkout as passing, ideally covering this workflow
as an additional in-test fixture case.

## Acceptance Criteria

- **AC-1**: Given a sample of measured three-call units where each unit's VLM extraction
  carries an explicit per-page call count, when the extrapolator computes the 50-case
  projection, then projected time/tokens/cost equal `50 ×` the per-unit aggregate with
  multi-page VLM calls counted explicitly and the retry-amplification factor applied —
  verified against hand-computed expectations on synthetic inputs.
- **AC-2**: Given known synthetic latency/token/cost arrays, when p50/p95 aggregation
  runs, then the results match hand-computed percentiles (including small-sample and
  tie edge cases).
- **AC-3**: Given a completed measurement run (synthetic/stubbed provider responses in
  tests), when the report is generated, then it contains every required field: sample
  size, per-call-class p50/p95 runtime, token counts, provider cost, rate-limit headroom,
  retry amplification, daily-quota statement, extrapolated 50-case totals, the named
  maximum per-run cost/time budget, a verdict ∈ {viable, stop_escalate} (closed set), and
  the W2-OA2 local-key substitution note field.
- **AC-4**: Given a run configured with key material in env and Authorization headers on
  outbound calls, when the report text is generated, then it contains **no key material**
  (no env secret values, no `sk-ant-` tokens) and **no Authorization header values** —
  a property test over adversarially seeded fake secrets.
- **AC-5**: Given the EXISTING `.github/workflows/*.yml` files (read-only), when the
  workflow policy lint runs, then **no workflow checks out fork PR code under
  `pull_request_target` with secrets access** — the lint parses trigger + checkout-ref +
  secret usage and fails on any violating combination (and is proven to fire on a
  synthetic violating workflow fixture provided in-test, not written to
  `.github/workflows/`) — while PASSING the known compliant near-miss
  `dependabot-auto-merge.yml` (`pull_request_target` + secrets but no PR-code checkout;
  see Context).
- **AC-6**: Given `docs/week2/W2_TIER2_CI_POLICY.md`, when the policy-doc lint runs, then
  the doc exists and states every frozen clause: no repository secrets to forks; never
  `pull_request_target` checkout of fork code; forks run Tier 1 only; maintainer
  reproduces the exact fork commit on a trusted same-repo branch for the required Tier-2
  result before merge; same-repo PRs use least-privilege environments with approval and
  no secret echo/artifact retention; and the STOP-escalation rule (quota/runtime/cost
  failure never reduces the 50 cases).
- **AC-7** [live-measure]: Given the local agent key from `agent/.env` (W2-OA2 pending —
  substitution noted in the output), when `agent/ops/spike_tier2.py` runs the
  representative sample (working size 5 units) of the real three-call-unit shape over
  runtime-generated synthetic non-clinical images, then measured runtime, tokens, cost,
  rate-limit headroom, and retry amplification are recorded; the 50-case extrapolation,
  the named max per-run cost/time budget, and the verdict on whether the full 50-case
  gate is viable as required are recorded in the ticket report — a failing fit is
  reported as **STOP escalation** (dependency problem), never a case-count reduction.
