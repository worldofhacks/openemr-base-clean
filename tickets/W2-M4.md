## Context

Every grounded field and bbox overlay downstream depends on one canonical coordinate
space. §2 (locked-decision): `NormBBox{x0,y0,x1,y1}` are normalized page-relative
coordinates ∈ [0,1], **origin top-left, y-down**. Both readers convert at ingestion —
Tesseract divides by rendered pixel dimensions; the PDF text layer divides by the media
box and **flips y** (pdfium/PDF space is y-up). Render DPI is **200** (locked) and page
pixel dimensions are recorded in the words+boxes layer. The reader stack is
**pypdfium2 + pdfplumber + Tesseract only — PyMuPDF is banned (AGPL, W2-R6)**.

Dependencies (pypdfium2, pdfplumber + its pillow dependency, pytesseract, and the
in-container tesseract binary) land with **W2-M1** — this ticket depends on it and must
NOT touch `agent/pyproject.toml` or `agent/Dockerfile`. The host has tesseract installed
(brew) for local test loops. `agent/app/ingestion/reader.py` (NEW) is the words+boxes
layer with both paths: text-layer first, a junk-text-layer **density heuristic** routing
to OCR (W2-D3), and Tesseract OCR at 200 DPI as the fallback; the OCR path renders pages
via pypdfium2. Per-page OCR runs under a **subprocess timeout**: a pathological page is
killed and marked unreadable with a typed outcome instead of hanging the job. Proposed:
`NormBBox` is defined in `reader.py` for this spike and unified into the canonical §2
contract module when W2-M6 freezes schemas — do not improvise a second shape afterward.
`agent/ops/spike_reader.py` (NEW) is the operator CLI that runs both paths over the
fixtures and prints the comparison evidence (ops scripts may print).

Fixtures: `agent/evals/fixtures/documents/` (NEW dir, expanded later by W2-M7) gets two
seed fixtures — one born-digital ("clean") PDF and one degraded scan-style fixture — plus
a **committed reproducible generator script** (e.g. `generate_fixtures.py` in the same
dir). Content is **synthetic and non-clinical only** (no PHI, no real-looking patient
data); the degraded fixture's noise is seeded so regeneration is deterministic (the
generator must pin PDF timestamps/metadata for stable bytes — proposed: hash-stable
regeneration; geometry-identical is the fallback assertion if byte-stability proves
infeasible, recorded in the report). Fixture files are not under the frozen-diff paths
(`.tdd-swarm/run-local-gates.sh` freezes `agent/tests/`, `agent/evals/test_*`,
`agent/ops/tests/`), so they may be authored during implementation — the fixtures dir
is therefore a **file scope** (implementer write authority), NOT a test scope; the
frozen tests reference the committed fixture paths. Residual risk — frozen tests
asserting over implementer-authored fixtures — is inherent to this design and accepted
for a spike: it is constrained by AC-7's reproducible-generator + synthetic-marker
check.

## Acceptance Criteria

- **AC-1**: Given the born-digital fixture and the same content rasterized for OCR, when
  the same word is extracted via the text-layer path (pypdfium2/pdfplumber, media-box
  normalization + y-flip) and via the OCR path (Tesseract at render DPI 200,
  pixel-dimension normalization), then both paths yield the **same NormBBox** for that
  word — equal within a small tolerance chosen by the test author and recorded in the
  test (proposed: two engines are never bit-equal; the tolerance is an explicit named
  constant, not a hidden fudge).
- **AC-2**: Given words+boxes output from both paths — the text-layer path, AND the OCR
  path run over the **committed degraded scan fixture** (Tesseract at render DPI 200),
  so the second seed fixture is load-bearing in a frozen test — when the geometry is
  validated, then every box satisfies the canonical-space invariants: all coordinates
  ∈ [0,1], `x0 < x1`, `y0 < y1`, origin top-left y-down — proven by a word known to sit
  near the top of the page yielding small y values on **both** paths (the PDF path
  demonstrably flipped y) — and the layer records render DPI (200) and page pixel
  dimensions.
- **AC-3**: Given a fixture whose embedded text layer is junk (garbage strings at
  plausible positions), when the reader's density heuristic evaluates it, then the page
  is routed to OCR and the words+boxes output records the OCR path as the source.
- **AC-4**: Given a pathological page whose OCR subprocess exceeds the per-page timeout
  (simulated deterministically via a seam — e.g. an injected fake/slow OCR runner), when
  the reader processes it, then the subprocess is killed, the page is marked unreadable
  with a typed outcome, remaining pages still process, and the test completes within
  bounded wall-clock — never a hang.
- **AC-5**: Given the installed reader stack, when shipped package metadata is inspected
  via `importlib.metadata`, then every shipped reader dependency reports a permissive
  license in the Apache/BSD/MIT family, **PyMuPDF is absent from the environment**, and
  no GPL/AGPL identifier appears; permissive-equivalent identifiers (e.g. pillow's
  HPND) are acceptable only via an explicit in-test allowlist entry with a justification
  comment (proposed).
- **AC-6** [live-measure]: The word-segmentation winner (**pypdfium2 vs pdfplumber**) is
  chosen on fixture evidence: `agent/ops/spike_reader.py` runs both candidates over the
  seed fixtures and the comparison (word counts, split/merge behavior, box quality) plus
  the decision and rationale are recorded in the ticket report.
- **AC-7**: Given the committed generator script run with its fixed seed, when its output
  is compared against the two committed seed fixtures, then they match (hash-stable; or
  the recorded geometry-identical fallback per Context) and fixture text contains only
  the known synthetic non-clinical content markers.
